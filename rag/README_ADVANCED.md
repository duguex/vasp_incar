## 🔧 配置参数

```python
config = {
    "json_file": "vasp_wiki_all_data.json",  # 数据文件
    "server_hosts": [...],                   # 服务器列表
    "max_workers": 6,                        # 并行工作数
    "chunk_size": 1000,                      # 分块大小
    "chunk_overlap": 200,                    # 重叠大小
    "persist_dir": "./chroma_db",            # 数据库目录
    "force_rebuild": False                   # 强制重建
}
```

### 并行工作数建议
- **2个服务器**: `max_workers = 2` (简单) 或 `4` (加速)
- **3个服务器**: `max_workers = 3` (简单) 或 `6` (加速)
- **4个服务器**: `max_workers = 4` (简单) 或 `8` (加速)

**说明**:
- 简单模式: `max_workers = 服务器数量` (1对1)
- 加速模式: `max_workers = 服务器数量 × 2` (充分利用)

## ⚡ 性能优化建议

### 1. 服务器选择
- 优先使用局域网服务器 (延迟低)
- 避免使用网络不稳定的服务器
- 定期检查服务器状态

### 2. 并行配置
```python
# 根据文档数量调整
if 文档数量 < 1000:
    max_workers = 2
elif 文档数量 < 5000:
    max_workers = 4
else:
    max_workers = 6
```

### 3. 分块策略
```python
# 根据内容复杂度调整
chunk_size = 800   # 技术文档，内容密集
chunk_overlap = 150
```

## 🔍 故障排除

### 问题1: 某些服务器离线
```bash
# 检查服务器状态
python test_servers.py

# 在配置中移除离线服务器
server_hosts = ["192.168.1.127", "127.0.0.1"]  # 只使用在线的
```

### 问题2: 嵌入生成慢
- 增加 `max_workers`
- 减少 `chunk_size`
- 检查网络延迟

### 问题3: 内存不足
- 减少 `max_workers`
- 减小 `chunk_size`
- 分批处理
